# Adaptive Learning AI

Adaptive Learning AI is a coursework project that combines:
- **BKT (Bayesian Knowledge Tracing)** for learner modeling
- **LLM-based generation** for lessons and question answering
- **RAG (Retrieval-Augmented Generation)** for grounding answers and lessons in course material

The current repository contains both the broader adaptive-learning app and an actively developed grounded-answer pipeline for student questions.

## Core Features

- learner state modeling with BKT
- course and module management from TOC-driven content
- document ingestion and chunking from textbook PDFs
- hybrid retrieval with:
  - dense vector search
  - BM25 sparse search
  - weighted reciprocal rank fusion
- query transformation strategies:
  - query expansion
  - query decomposition
  - step-back prompting
  - HyDE
- reranking before final answer generation
- grounded answer generation with source-aware citations
- Streamlit interface for interactive workflows

## Project Structure

- `app/`
  Streamlit UI and interactive app flow.
- `core/`
  Core learner, BKT, engine, and shared data models.
- `llm/`
  LLM-related modules such as query transforms, reranker, lesson generation, and answer prompt building.
- `rag/`
  Document processing, chunking, retrieval, vector store, grounding, and answer orchestration.
- `services/`
  Higher-level orchestration for content, lessons, and learner state.
- `data/`
  Courses, skills, TOCs, indexes, cached lessons, and learner data.
- `config/`
  Application settings.
- `utils/`
  Utility helpers.

## Current Answer Pipeline

For a student question, the main grounded-answer path is:

1. student enters a query
2. query transform selector chooses one strategy
3. query is transformed if needed
4. hybrid retrieval collects candidate chunks
5. reranker selects the best chunks
6. grounded context is built from those chunks
7. answer prompt is built from grounded sources and grounded context
8. final answer is generated by the LLM

## Setup

### 1. Create a virtual environment

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

### 2. Install dependencies

```powershell
pip install -r requirements.txt
```

### 3. Configure environment variables

Create a local `.env` file and set:

```text
OPENAI_API_KEY=your_openai_api_key_here
```

## Run the Project

### Main app

```powershell
python main.py
```

### Streamlit UI

```powershell
streamlit run app/streamlit_app.py
```

### Debug the grounded answer pipeline

```powershell
python debug_answer.py
```

### Debug query transform and retrieval

```powershell
python debug_query_transform.py
```

## Notes

- `.env` is ignored and should not be committed.
- The repo may contain cached/generated files under `data/` depending on prior runs.
- Cross-encoder reranking requires local model dependencies to be installed.
- If the cross-encoder stack is unavailable, reranking falls back to a simpler lexical strategy.

## Status

This repository is an actively iterated project, so some modules are more complete than others. The grounded-answer pipeline under `llm/` and `rag/` is the most actively refined part at the moment.
